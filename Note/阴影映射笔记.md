# 阴影映射（Shadow Mapping）

> 参考：[haqr.eu - Shadows](https://haqr.eu/tinyrenderer/shadows/) / Two-Pass Shadow Rendering

---

## 一、核心思想

阴影映射的基本原理：**从光源视角渲染一次深度图，再从相机视角渲染时，将每个片元投影回光源空间，比较深度判断是否被遮挡**。

```
Pass 1: 从光源视角渲染 → 生成 Shadow Map（深度缓冲）
Pass 2: 从相机视角渲染 → 对每个片元查询 Shadow Map → 判断阴影
```

---

## 二、两遍渲染流程

### 2.1 第一遍：光源深度渲染（Shadow Pass）

```cpp
// 以光源位置为"相机"，构建 MVP
Vec3f light(0.5, 1, 1);
ModelView(light, center, up);                          // 光源作为 eye
Perspective((light - center).norm());                   // 光源焦距
Viewport(width/16, height/16, width*7/8, height*7/8);

// 保存光源空间的完整变换矩阵
Mat4 M_Light = viewport * perspective * modelview;

// 用 MapShader 渲染，仅写深度，不关心颜色
zbufferf.assign(width * height, -1e10f);
TGAImage shadowImg(width, height, TGAImage::RGB);
for (int m = 0; m < model_count; m++) {
    MapShader shadowShader(&models[m]);
    for (int i = 0; i < models[m].nfaces(); i++) {
        Vec4f face[3] = {shadowShader.Vertex(i,0), shadowShader.Vertex(i,1), shadowShader.Vertex(i,2)};
        Rasterize(shadowImg, face, shadowShader);
    }
}

// 保存光源深度缓冲
std::vector<float> shadowBuffer = zbufferf;
```

### 2.2 MapShader — 光源深度专用 Shader

```cpp
class MapShader : public IShader {
public:
    const Model *model;

    MapShader(const Model *model) : model(model) {}

    virtual Vec4f Vertex(const int face, const int vert) {
        Vec4f v = {model->vert(model->face(face)[vert]), 1};
        return perspective * modelview * v;
    }

    virtual std::pair<bool, TGAColor> fragment(const Vec3f bar) const {
        TGAColor color{0, 0, 0, 0};
        return {false, color};
    }
};
```

**要点**：
- `Vertex()` 执行标准 MVP 变换，`Rasterize` 内部自动更新全局 `zbufferf`
- `fragment()` 返回空颜色——第一遍只关心深度，不需要颜色输出

### 2.3 第二遍：相机渲染 + 阴影测试（Eye Pass）

```cpp
// 恢复相机视角的 MVP
ModelView(eye, center, up);
Perspective((eye - center).norm());
Viewport(width/16, height/16, width*7/8, height*7/8);

// 计算眼空间→光源空间的变换矩阵
Mat4 M_Shadow = M_Light * inverse(modelview);

// 重置深度缓冲，用 PhongShader 渲染
zbufferf.assign(width * height, -1e10f);
for (int m = 0; m < model_count; m++) {
    PhongShader shader(&models[m], light, shadowBuffer, M_Shadow, width, height);
    for (int i = 0; i < models[m].nfaces(); i++) {
        Vec4f face[3] = {shader.Vertex(i,0), shader.Vertex(i,1), shader.Vertex(i,2)};
        Rasterize(img, face, shader);
    }
}
```

## 三、M_Shadow 矩阵的推导

**目标**：将眼空间中的片元位置变换到光源空间，以便查询 shadowBuffer。

**推导**：

```
光源空间坐标 = M_Light × 模型空间坐标
眼空间坐标   = modelview × 模型空间坐标

因此：
  模型空间坐标 = inverse(modelview) × 眼空间坐标

代入第一式：
  光源空间坐标 = M_Light × inverse(modelview) × 眼空间坐标
               = M_Shadow × 眼空间坐标
```

```cpp
Mat4 M_Shadow = M_Light * inverse(modelview);
```

**注意**：`M_Light = viewport * perspective * modelview_light`，包含了完整的 MVP+Viewport 变换，因此 `M_Shadow` 将眼空间坐标直接映射到光源视角的**屏幕像素坐标**（含深度），可直接用来索引 `shadowBuffer`。

## 四、PhongShader 中的阴影测试

### 4.1 新增成员

```cpp
class PhongShader : public IShader {
public:
    // ... 原有成员 ...
    std::vector<float> shadowBuffer;   // 光源深度缓冲
    Mat4 M_shadow;                      // 眼空间→光源屏幕空间的变换矩阵
    int shadow_width;                   // shadow map 宽度
    int shadow_height;                  // shadow map 高度

    PhongShader(const Model *model, Vec3f light,
                std::vector<float> sb, Mat4 ms, int w, int h)
        : model(model), shadowBuffer(sb), M_shadow(ms),
          shadow_width(w), shadow_height(h) {
        l = ((modelview * Vec4f(light, 0)).toVec3()).normalize();
    }
};
```

### 4.2 fragment() 中的阴影计算

```cpp
// 1. 插值得到当前片元在眼空间的位置
Vec4f p_view = tri[0] * bar.x + tri[1] * bar.y + tri[2] * bar.z;

// 2. 变换到光源屏幕空间
Vec4f p_light = M_shadow * p_view;
p_light = p_light / p_light.w;    // 透视除法

// 3. 阴影判定
float shadow_diff = 1.f;    // 默认不遮挡
float shadow_spec = 1.f;
float bias = 0.05f;         // 深度偏移，防止 shadow acne

if (p_light.x >= 0 && p_light.x < shadow_width &&
    p_light.y >= 0 && p_light.y < shadow_height) {
    int idx_x = (int)p_light.x;
    int idx_y = (int)p_light.y;
    int shadow_idx = idx_x + idx_y * shadow_width;
    if (p_light.z + bias < shadowBuffer[shadow_idx]) {
        shadow_diff = 0.05f;    // 漫反射几乎归零
        shadow_spec = 0.f;      // 高光完全消除
    }
}
```

**逐行解析**：

| 步骤 | 代码 | 说明 |
|------|------|------|
| 1 | `p_view = tri[0]*bar.x + ...` | 用重心坐标插值眼空间顶点位置 |
| 2 | `p_light = M_shadow * p_view` | 变换到光源屏幕空间 |
| 3 | `p_light / p_light.w` | 透视除法，得到归一化坐标 |
| 4 | 边界检查 | 确保 `(idx_x, idx_y)` 在 shadow map 范围内 |
| 5 | `p_light.z + bias < shadowBuffer[...]` | 当前片元深度 + 偏移 < 光源记录深度 → 在阴影中 |

### 4.3 阴影对光照的影响

```cpp
double amb  = 0.35;
double diff = std::max(0.f, n * l);
double spec = std::pow(std::max(0.f, r * v_dir), 40);
for (int i = 0; i < 3; i++) {
    color[i] *= std::min(1., amb + 2 * shadow_diff * diff + shadow_spec * 2 * spec);
}
```

| 光照分量 | 不在阴影中 | 在阴影中 |
|----------|-----------|---------|
| **Ambient** | 0.35 | 0.35（不受阴影影响） |
| **Diffuse** | `2.0 × 1.0 × diff` = `2 × diff` | `2.0 × 0.05 × diff` ≈ `0.1 × diff` |
| **Specular** | `2.0 × 1.0 × spec` = `2 × spec` | `2.0 × 0 × spec` = `0` |

阴影中漫反射被压缩到约 5%，高光完全消失，但环境光保留——模拟真实世界中阴影区域仍有的微弱环境照明。

## 五、深度偏移（Depth Bias）

```cpp
float bias = 0.05f;
if (p_light.z + bias < shadowBuffer[shadow_idx]) { ... }
```

### 5.1 为什么需要 Bias？

由于浮点精度限制，同一点的深度在两遍渲染中可能有微小差异。不加 bias 时：

```
p_light.z = 5.0000001    （第二遍计算值）
shadowBuffer[...] = 4.9999999   （第一遍记录值）

p_light.z > shadowBuffer → 判定"被遮挡" → 误判为阴影！
```

这会产生 **Shadow Acne**（阴影痤疮）：物体表面出现条纹状的自阴影伪影。

### 5.2 Bias 的工作方式

加入 `bias = 0.05` 后，将当前深度稍微推远一点：

```
p_light.z + bias = 5.05 > shadowBuffer → 判定"不在阴影中" ✓
```

只有真正被其他物体遮挡的点（深度差远大于 bias）才会被判为阴影。

### 5.3 Bias 取值权衡

| bias 值 | 效果 |
|---------|------|
| 太小 | Shadow Acne（自阴影伪影） |
| 0.05 | 当前取值，平衡效果 |
| 太大 | Peter Panning（阴影脱离物体，悬浮在地面之上） |

## 六、整体渲染流程（含阴影）

```
┌──────────────────────────────────────────────────────────────────────────┐
│                     阴影映射渲染流程                                      │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Pass 1 — 光源深度渲染                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │ ModelView(light, center, up)          ← 光源作为相机               │ │
│  │ Perspective + Viewport                                                 │ │
│  │ M_Light = viewport * perspective * modelview   ← 保存变换矩阵      │ │
│  │ 对每个模型 × 每个面:                                                   │ │
│  │   MapShader.Vertex() → Rasterize() → 写入 zbufferf                  │ │
│  │ shadowBuffer = zbufferf           ← 保存光源深度                    │ │
│  └─────────────────────────────────────────────────────────────────────┘ │
│                                                                          │
│  Pass 2 — 相机渲染 + 阴影测试                                           │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │ ModelView(eye, center, up)           ← 恢复相机视角                │ │
│  │ M_Shadow = M_Light * inverse(modelview)  ← 眼空间→光源空间矩阵    │ │
│  │ 重置 zbufferf                                                          │ │
│  │ 对每个模型 × 每个面:                                                   │ │
│  │   PhongShader.Vertex() → Rasterize()                                  │ │
│  │     └─ fragment():                                                     │ │
│  │        ├─ TBN + 法线贴图 + 漫反射贴图（同前）                         │ │
│  │        ├─ p_view → M_Shadow → p_light  ← 变换到光源空间             │ │
│  │        ├─ 比较 p_light.z + bias vs shadowBuffer → 阴影判定          │ │
│  │        └─ shadow_diff/shadow_spec 调制光照                             │ │
│  └─────────────────────────────────────────────────────────────────────┘ │
│                                                                          │
│  后处理: ApplySSAO(img, zbufferf, width, height)                         │
│  输出: output.tga + zbuffer.tga                                          │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

## 七、阴影映射的局限性

| 问题 | 说明 |
|------|------|
| **分辨率限制** | Shadow Map 分辨率与屏幕相同，远处物体阴影可能出现锯齿（Aliasing） |
| **固定 Bias** | 当前 bias=0.05 为固定值，不同场景可能需要调整 |
| **硬阴影** | 只产生硬阴影（二值判定：在/不在阴影中），无半影（Penumbra） |
| **光源类型** | 仅支持平行光/点光源，聚光灯需要额外处理 |
| **Peter Panning** | bias 过大时阴影可能脱离物体 |

### 可能的改进方向

1. **PCF（Percentage Closer Filtering）**：对 shadow map 做多次采样取平均，产生软阴影
2. **CSM（Cascaded Shadow Maps）**：对近处和远处使用不同分辨率的 shadow map
3. **自适应 Bias**：基于表面法线和光照方向的夹角动态计算 bias
4. **VSM（Variance Shadow Maps）**：存储深度的均值和方差，用概率方法做软阴影
