## 一、透视校正插值（Perspective-Correct Interpolation）

### 1.1 问题

光栅化时，重心坐标在**屏幕空间**计算，但 varying 变量（UV、法线等）定义在**3D 空间**。直接用屏幕空间重心坐标插值 3D 属性，在透视投影下会产生失真。

**直观例子**：一条直线在 3D 中被等分，透视投影后在屏幕上近处段长、远处段短。如果直接用屏幕重心坐标插值 UV，近处的纹理会被拉伸、远处被压缩。

### 1.2 数学推导

设三个顶点的 clip-space 坐标为 `(x_i, y_i, z_i, w_i)`，某 varying 属性为 `A_i`。

屏幕空间重心坐标 `(α, β, γ)` 是在透视除法之后计算的。要正确插值 3D 属性，需要先除以 `w`，插值后再乘回来：

```
A_interp = (α × A₀/w₀ + β × A₁/w₁ + γ × A₂/w₂)
          / (α/w₀ + β/w₁ + γ/w₂)
```

等价写法：令 `bar' = (α/w₀, β/w₁, γ/w₂)`，然后 `bar' = bar' / (bar'.x + bar'.y + bar'.z)`，则：

```
A_interp = bar'.x × A₀ + bar'.y × A₁ + bar'.z × A₂
```

### 1.3 代码实现

```cpp
// renderer.cpp — Rasterize() 中
Vec3f heavyPoint = invert_transpose(area) * Vec3f(x, y, 1);  // 屏幕空间重心坐标
if (heavyPoint.x < 0 || heavyPoint.y < 0 || heavyPoint.z < 0) continue;

// 透视校正：除以各顶点的 w 分量
Vec3f bar = Vec3f(heavyPoint.x / face[0].w,
                  heavyPoint.y / face[1].w,
                  heavyPoint.z / face[2].w);
bar = bar / (bar.x + bar.y + bar.z);    // 重新归一化

auto [discard, color] = shader.fragment(bar);  // 用校正后的重心坐标调用 fragment
```

**对比**：

| | 旧版（线性插值） | 新版（透视校正） |
|---|---|---|
| 重心坐标 | `bar = heavyPoint` | `bar = (α/w₀, β/w₁, γ/w₂) / sum` |
| UV 插值 | 近处拉伸、远处压缩 | 正确的透视映射 |
| 法线插值 | 可能出现偏差 | 更准确的光照 |

### 1.4 为什么 Z 值不用校正？

```cpp
float z = heavyPoint * Vec3f{ndc[0].z, ndc[1].z, ndc[2].z};  // 仍然用原始重心坐标
```

深度值 `z` 本身已经是透视除法后的 NDC 深度，在屏幕空间是线性的（这是透视投影的数学性质），所以用原始 `heavyPoint` 插值 `z` 是正确的。

## 二、TBN Gram-Schmidt 正交化

### 2.1 问题

从 `T = U⁻¹ · E` 计算出的切线 T 和副切线 B，由于数值精度和浮点误差，通常不是精确正交的：

- T 和 B 可能不垂直于 N
- T 和 B 可能不互相垂直

如果直接归一化后组装 TBN 矩阵，法线贴图变换会产生偏差。

### 2.2 Gram-Schmidt 正交化

**目标**：使 T、B、N 互相正交且单位长度。

**步骤**：

```
1. N 优先级最高（来自顶点法线插值，最可靠）
2. T 去除 N 分量，投影到 N 的切平面上
3. B 去除 N 和 T 的分量，确保与两者正交
```

### 2.3 代码实现

```cpp
// 提取 T 和 B 的 3D 分量
Vec3f t = {T[0][0], T[0][1], T[0][2]};    // Tangent
Vec3f b = {T[1][0], T[1][1], T[1][2]};    // Bitangent

// 插值法线
Vec3f a2 = varying_normal[0]*bar.x + varying_normal[1]*bar.y + varying_normal[2]*bar.z;
a2.normalize();                            // N

// Step 1: T 投影到 N 的切平面
t = t - a2 * (t * a2);                    // 去除 N 方向分量
t.normalize();

// Step 2: B 投影到 N 和 T 张成的平面的补空间
b = b - a2 * (b * a2);                    // 去除 N 方向分量
b = b - t * (b * t);                      // 去除 T 方向分量
b.normalize();

// 组装正交化的 TBN 矩阵
Mat4 A;
A[0] = {t.x, t.y, t.z, 0};
A[1] = {b.x, b.y, b.z, 0};
A[2] = {a2.x, a2.y, a2.z, 0};
A[3] = {0, 0, 0, 1};
```

### 2.4 对比

| | 旧版（仅归一化） | 新版（Gram-Schmidt） |
|---|---|---|
| T | `a0.normalize()` | `t = t - a2*(t*a2); t.normalize()` |
| B | `a1.normalize()` | `b = b - a2*(b*a2) - t*(b*t); b.normalize()` |
| 正交性 | 不保证 | 保证 T⊥N、B⊥N、B⊥T |
| 法线贴图质量 | 可能有方向偏差 | 更准确 |

### 2.5 数学原理

Gram-Schmidt 正交化的核心：`v_⊥ = v - proj_u(v) = v - (v·u)u`

- `t = t - (t·n)n`：去除 T 在 N 方向的投影，使 T 躺在 N 的切平面上
- `b = b - (b·n)n - (b·t')t'`：去除 B 在 N 和 T 方向的投影，使 B 与两者正交

## 三、真实视线方向高光

### 3.1 旧版简化

```cpp
// 旧版：假设视线方向始终为 (0, 0, -1)
double spec = std::pow(std::max(0.f, r.z), 40);
```

这在眼空间中，当相机正对画面时是正确的（视线沿 -Z），但对于画面边缘的片元，视线方向并非精确的 `(0, 0, -1)`。

### 3.2 新版：计算真实视线方向

```cpp
// 当前片元在眼空间的位置
Vec4f p_view = tri[0] * bar.x + tri[1] * bar.y + tri[2] * bar.z;

// 视线方向：从片元指向相机（原点）
Vec3f v_dir = -p_view.toVec3().normalize();

// 用真实视线方向计算高光
double spec = std::pow(std::max(0.f, r * v_dir), 40);
```

**原理**：在眼空间中，相机位于原点 `(0, 0, 0)`，片元位置为 `p_view`。从片元看向相机的方向为 `-p_view`。

### 3.3 改进效果

| | 旧版 | 新版 |
|---|---|---|
| 视线方向 | 固定 `(0,0,-1)` | 从片元到相机的真实方向 |
| 画面中心 | 正确 | 正确 |
| 画面边缘 | 高光位置有偏差 | 更精确 |
| `r * v_dir` | 简化为 `r.z` | 完整点积 |

## 四、阴影感知的 Phong 光照

### 4.1 完整光照公式

```cpp
double amb  = 0.35;                                    // 环境光
double diff = std::max(0.f, n * l);                    // 漫反射
double spec = std::pow(std::max(0.f, r * v_dir), 40); // 高光

color[i] *= std::min(1., amb + 2 * shadow_diff * diff + shadow_spec * 2 * spec);
```

### 4.2 光照系数对比

| 分量 | 旧版系数 | 新版系数 | 说明 |
|------|---------|---------|------|
| Ambient | 0.3 | 0.35 | 略微增加环境光 |
| Diffuse | 0.4 | `2 × shadow_diff` | 默认 shadow_diff=1，系数=2；阴影中 shadow_diff=0.05，系数≈0.1 |
| Specular | 0.9 | `2 × shadow_spec` | 默认 shadow_spec=1，系数=2；阴影中 shadow_spec=0，系数=0 |
| Specular 指数 | 40 | 40 | 不变 |

### 4.3 阴影对三个分量的影响

```
               不在阴影中              在阴影中
Ambient:       0.35                   0.35         ← 不受影响
Diffuse:       2.0 × diff             0.1 × diff   ← 压缩到 5%
Specular:      2.0 × spec             0            ← 完全消除
```

设计理念：阴影区域仍有环境光照明（不会完全变黑），但漫反射和高光被大幅抑制。

## 五、多模型渲染

### 5.1 输入方式

```cpp
int model_count;
std::cin >> model_count;

std::vector<std::string> paths(model_count);
for (int i = 0; i < model_count; i++) {
    std::cin >> paths[i];
}

std::vector<Model> models;
for (int i = 0; i < model_count; i++) {
    models.emplace_back(paths[i]);
}
```

### 5.2 两遍渲染中的多模型

**Shadow Pass**：遍历所有模型，统一写入光源深度

```cpp
for (int m = 0; m < model_count; m++) {
    MapShader shadowShader(&models[m]);
    for (int i = 0; i < models[m].nfaces(); i++) {
        Vec4f face[3] = {shadowShader.Vertex(i,0), shadowShader.Vertex(i,1), shadowShader.Vertex(i,2)};
        Rasterize(shadowImg, face, shadowShader);
    }
}
```

**Eye Pass**：遍历所有模型，使用统一的 shadowBuffer

```cpp
for (int m = 0; m < model_count; m++) {
    PhongShader shader(&models[m], light, shadowBuffer, M_Shadow, width, height);
    for (int i = 0; i < models[m].nfaces(); i++) {
        Vec4f face[3] = {shader.Vertex(i,0), shader.Vertex(i,1), shader.Vertex(i,2)};
        Rasterize(img, face, shader);
    }
}
```

**关键**：所有模型共享同一个 `zbufferf` 和 `shadowBuffer`，因此模型之间的遮挡关系和阴影关系是正确的。

## 六、TGAImage 填充色构造函数

### 6.1 新增签名

```cpp
TGAImage(const int w, const int h, const int bpp, TGAColor c = {});
```

创建指定尺寸的图像，并用颜色 `c` 填充所有像素。默认参数 `c = {}` 即全透明黑色 `{0,0,0,0}`，保持与旧版行为兼容。

### 6.2 使用示例

```cpp
// 旧版：黑色背景
TGAImage img(width, height, TGAImage::RGB);

// 新版：天蓝色背景
TGAImage img(width, height, TGAImage::RGB, {177, 195, 209, 255});
```

颜色 `{177, 195, 209, 255}` 是 BGRA 顺序，对应 RGB 的 `(209, 195, 177)`，一种柔和的天蓝色。

## 七、CreateZBufferImage — 深度可视化

### 7.1 函数签名

```cpp
TGAImage CreateZBufferImage(const std::vector<float>& zbufferf, int width, int height);
```

### 7.2 实现逻辑

```cpp
// 1. 找到深度值的有效范围（排除背景 -1e10f）
float zmin = ..., zmax = ...;
for (int i = 0; i < width * height; i++) {
    if (zbufferf[i] > -1e9f) {
        zmin = std::min(zmin, zbufferf[i]);
        zmax = std::max(zmax, zbufferf[i]);
    }
}

// 2. 线性映射到 [0, 255] 灰度
TGAImage zimg(width, height, TGAImage::GRAYSCALE);
for (int x = 0; x < width; x++) {
    for (int y = 0; y < height; y++) {
        float z = zbufferf[x + y * width];
        uint8_t val = (uint8_t)((z - zmin) / (zmax - zmin) * 255);
        zimg.set(x, y, TGAColor{val});
    }
}
```

**要点**：
- 背景（`z < -1e9f`）不参与范围计算，映射后为 0（黑色）
- 有效深度线性映射到 0-255 灰度
- 白色 = 最近，黑色 = 最远或背景
