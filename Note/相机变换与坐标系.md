# 相机变换与坐标系（Better Camera Handling）

> 参考：[haqr.eu - Better camera handling](https://haqr.eu/tinyrenderer/camera/)

---

## 一、坐标变换链（Chain of Coordinate Transformations）

标准渲染管线中，一个从 `.obj` 读取的顶点 `v` 需要经过五级坐标系、四次变换：

```
Object Space → World Space → Eye/Camera Space → Clip Space → Screen Space
        Model               View              Perspective         Viewport
```

用函数表示即：`Viewport(Perspective(View(Model(v))))`

**✅ 当前项目已实现**：所有四步变换均已迁移到矩阵乘法，统一为 `clip = Perspective × ModelView × Vec4(v, 1)`。

---

## 二、线性变换（Linear Transformations）

在 ℝⁿ 中，线性变换可写成 `x ↦ A·x`，其中 A 是 n×n 矩阵。它们保持直线、保持原点不动。

| 变换 | 2D 矩阵 | 说明 |
|------|---------|------|
| **Identity（恒等）** | `[1 0; 0 1]` | 不改变任何点 |
| **Scaling（缩放）** | `[sx 0; 0 sy]` | 各轴独立伸缩 |
| **Rotation（旋转）** | `[cosθ -sinθ; sinθ cosθ]` | 绕原点逆时针旋转 θ |
| **Shear（剪切）** | `[1 sh; 0 1]` | 像果冻一样倾斜 |

**可组合性**：矩阵乘法满足结合律，因此 `A₃(A₂(A₁·x)) = ((A₃·A₂)·A₁)·x`。可以预先计算 `A = A₃·A₂·A₁`，然后一次性应用到每个顶点上。

---

## 三、仿射变换（Affine Transformations）

线性变换始终将原点映射到原点。图形学中常需要**平移（Translation）**，因此引入仿射变换：

```
T(x) = A·x + b
```

其中 `A` 是线性变换矩阵，`b` 是平移向量。但两个仿射变换的组合会变得非常冗长，不利于连续复合。

---

## 四、齐次坐标（Homogeneous Coordinates）

**核心技巧**：把 2D 点 `(x, y)` 嵌入到 3D 的平面 `z = 1` 上，写成 `(x, y, 1)`；3D 点则嵌入到 4D `(x, y, z, 1)`。

这样，仿射变换可以写成单一的矩阵乘法：

```
[ A  b ]   [ x ]   [ A·x + b ]
[ 0  1 ] · [ 1 ] = [    1    ]
```

齐次坐标的本质：一个点 `(x, y)` 可以表示为任意 `(w·x, w·y, w)`（w ≠ 0），恢复 Cartesian 坐标时做除法：`(x/w, y/w)`。

**两大优势**：
1. **平移变成矩阵乘法** —— 所有变换统一为矩阵，方便复合。
2. **天然支持透视除法** —— 投影只需除以 `w` 分量，相机处理（旋转、缩放、平移、投影）全部在同一框架内完成。

这也是为什么 2D 图形常用 3×3 矩阵、3D 图形常用 4×4 矩阵的原因。

**✅ 项目中的实现**：`geometry.h` 中 `Vec4` 类完整支持齐次坐标操作：
- `Vec4(Vec3f, 1)` — 表示点（w=1），平移有效
- `Vec4(Vec3f, 0)` — 表示方向向量（w=0）
- `toVec3()` — 执行透视除法 `(x/w, y/w, z/w)`；w=0 时直接截取 xyz
- `4×4 矩阵 × Vec3` — 自动视为 `(x,y,z,1)` 并执行透视除法

## 五、三大矩阵：Viewport / Perspective / ModelView

### ✅ 5.1 Viewport 矩阵（视口变换）— 已实现

```cpp
void Viewport(int x, int y, int w, int h) {
    viewport[0] = {w/2.f, 0, 0, x+w/2.f};
    viewport[1] = {0, h/2.f, 0, y+h/2.f};
    viewport[2] = {0, 0, 1, 0};
    viewport[3] = {0, 0, 0, 1};
}
```

将 NDC 的 `[-1, 1]` 映射到屏幕上以 `(x, y)` 为左上角、`(w, h)` 为尺寸的窗口。

数学形式：
```
[ w/2   0    0   x+w/2 ]
[  0   h/2   0   y+h/2 ]
[  0    0    1     0   ]
[  0    0    0     1   ]
```

### ✅ 5.2 Perspective 矩阵（透视变形）— 已实现

```cpp
void Perspective(float f) {
    perspective[0] = {1, 0, 0, 0};
    perspective[1] = {0, 1, 0, 0};
    perspective[2] = {0, 0, 1, 0};
    perspective[3] = {0, 0, -1/f, 1};
}
```

对点 `(vx, vy, vz, 1)` 做乘法后得到 `(vx, vy, vz, 1 - vz/f)`，再做透视除法（除以 w）即：
```
(vx, vy, vz) / (1 - vz/f)
```

`f` 是焦距，通常取 `(eye - center).norm()`。

### ✅ 5.3 ModelView 矩阵（相机/视图变换）— 已实现

灵感来自 OpenGL 的 `gluLookAt(eye, center, up)`：

```cpp
void ModelView(const Vec3f eye, const Vec3f center, const Vec3f up) {
    Vec3f n = (eye - center).normalize();   // 相机前向（视线反方向）
    Vec3f l = (up ^ n).normalize();         // 相机右向
    Vec3f m = (n ^ l).normalize();          // 相机上向

    Mat4 rot = {
        {l.x, l.y, l.z, 0},
        {m.x, m.y, m.z, 0},
        {n.x, n.y, n.z, 0},
        {0,   0,   0,   1}
    };
    Mat4 trans = {
        {1, 0, 0, -center.x},
        {0, 1, 0, -center.y},
        {0, 0, 1, -center.z},
        {0, 0, 0, 1}
    };
    modelview = rot * trans;
}
```

**参数**：
- `eye`：相机位置（世界坐标）
- `center`：相机看向的目标点
- `up`：大致的上方向

**坐标变换原理**：把世界坐标 `(x, y, z)` 变换到相机坐标 `(x', y', z')`：

1. 先平移，使相机原点到世界原点：`(-center)`
2. 再做基变换，把世界基 `(i, j, k)` 换成相机基 `(l, m, n)`

由于 `(i, j, k)` 是单位阵，基变换矩阵 `M` 就是 `[l m n]`（三列分别是 l, m, n）。而 `M` 是正交矩阵，所以 `M⁻¹ = Mᵀ`。

最终 ModelView 矩阵：
```
[ lx  ly  lz  0 ]   [ 1  0  0  -cx ]   [ lx  ly  lz  -l·c ]
[ mx  my  mz  0 ] · [ 0  1  0  -cy ] = [ mx  my  mz  -m·c ]
[ nx  ny  nz  0 ]   [ 0  0  1  -cz ]   [ nx  ny  nz  -n·c ]
[ 0   0   0   1 ]   [ 0  0  0   1  ]   [ 0   0   0    1   ]
```

## 六、✅ 最终渲染流程（矩阵版）— 已实现

初始化三个全局 4×4 矩阵：

```cpp
Mat4 modelview, perspective, viewport;

// 主函数中
ModelView(eye, center, up);                      // 构建 ModelView
Perspective((eye - center).norm());               // 构建 Perspective
Viewport(width/16, height/16, width*7/8, height*7/8);  // 构建 Viewport
```

对每个三角形的三个顶点，在 Shader 中：

```cpp
virtual Vec4f Vertex(const int face, const int vert) {
    Vec4f v = {model->vert(model->face(face)[vert]), 1};
    tri[vert] = (modelview * v).toVec3();    // 保存相机空间坐标（供 fragment 使用）
    return perspective * modelview * v;       // 返回 Clip Space 坐标
}
```

然后在 `Rasterize` 中：

```cpp
Vec4f ndc[3] = {
    Vec4f(face[0].x/face[0].w, face[0].y/face[0].w, face[0].z/face[0].w, 1),
    Vec4f(face[1].x/face[1].w, face[1].y/face[1].w, face[1].z/face[1].w, 1),
    Vec4f(face[2].x/face[2].w, face[2].y/face[2].w, face[2].z/face[2].w, 1),
};                                              // 透视除法 → NDC
for (int i=0; i<3; i++) ndc[i] = viewport * ndc[i];   // 视口映射 → 屏幕坐标

// 用 3×3 矩阵求重心坐标，做 Z-test，调用 fragment shader
```

**相比旧版实现的改进**：

| 项目 | 旧版实现 | 新版实现（当前） |
|------|---------|---------------|
| 模型旋转 | `rotation_y()` 硬编码 | 并入 `ModelView`，通过 lookat 自由定位相机 |
| 透视 | `persp()` 简化近似 | `Perspective` 矩阵，标准透视除法 |
| 视口映射 | `project()` 手算 | `Viewport` 矩阵，支持子窗口 |
| 相机 | 无相机概念，固定视角 | `ModelView()` 像 OpenGL 一样自由摆放相机 |
| 变换复合 | 每顶点三次函数调用 | 一次矩阵乘法 `Perspective × ModelView` |
| Z 缓冲 | TGAImage 灰度（8-bit） | `std::vector<float>` 浮点缓冲 |
| 着色方式 | 固定随机颜色 | `IShader` 抽象接口 |

## 七、重心坐标的矩阵法（当前实现）

旧版用面积比 `alpha = Area(P, B, C) / Area(A, B, C)` 逐个计算重心坐标。新版改为矩阵法：

```cpp
// 将 3 个屏幕空间顶点排成 3×3 矩阵
Mat3 area;
area[0] = {screen[0].x, screen[0].y, 1};
area[1] = {screen[1].x, screen[1].y, 1};
area[2] = {screen[2].x, screen[2].y, 1};

// 一次求出所有重心坐标
Vec3f heavyPoint = invert_transpose(area) * Vec3f(x, y, 1);
// heavyPoint = (α, β, γ)
```

**数学原理**：

三角形内点 `P = α·A + β·B + γ·C`，展开为：
```
P.x = α·A.x + β·B.x + γ·C.x
P.y = α·A.y + β·B.y + γ·C.y
1   = α      + β      + γ
```

写成矩阵形式：`area × (α, β, γ)ᵀ = (x, y, 1)ᵀ`

因此：`(α, β, γ)ᵀ = area⁻ᵀ × (x, y, 1)ᵀ`

**优势**：一次矩阵运算同时算出三个重心坐标分量，避免重复面积计算。

## 八、Shader 接口设计

`IShader` 抽象了渲染管线中的可编程阶段：

```cpp
class IShader {
public:
    virtual Vec4f Vertex(const int face, const int vert) = 0;    // 顶点着色器
    virtual std::pair<bool, TGAColor> fragment(const Vec3f bar) const = 0;  // 片元着色器
};
```

- **Vertex 着色器**：接收面索引和顶点索引，返回 MVP 变换后的 Clip Space 坐标
- **Fragment 着色器**：接收重心坐标，返回 `{是否丢弃, 颜色}`，可用于纹理采样、光照计算等

**当前 RandomShader** 在 Vertex 中额外保存相机空间顶点 `tri[vert]`，为后续光照计算做准备。

## 九、后续扩展方向

1. **Model 矩阵分离**：将模型自身的旋转/缩放从 ModelView 中独立，形成 `Model × View × Projection` 三矩阵
